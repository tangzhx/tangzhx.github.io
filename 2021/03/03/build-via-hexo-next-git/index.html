hexo/next + gitee/github搭建博客前

Posted on 2021-03-03

前提

Hexo 基于 Node.js 的静态博客框架，通过把 Markdown 格式的文章渲染成静态 Html 网页，通过 Git 进行文章的发布、更改，可以部署到任意的 Git 平台（Github、Gitlab、Coding、Gitee…）以及自己的服务器 / VPS。以下命令行操作在 Win10 Terminal 的 WSL ubuntu20.04 界面执行。

配置nodejs 和 Git

在开始前我们需要安装以下两个软件：

  • Node.js (推荐 LTS 版)
  • Git

安装好之后在cmd中输入npm -v和node -v，如果顺利打印出版本号，则说明node.js安装成功。回到桌面，单击鼠标右键，如果出现下图红框中的选项，则说明Git也安装成功了。

Github 仓库

  • 创建一个仓库，仓库设置为公开的才能免费使用Github Pages服务。
  • 仓库名必须设置为：**<用户名>.github.io**
  • 我的Github用户名为tangzhixiong，那么就需要创建一个名为tangzhixiong.github.io的公开仓库。
  • 创建完成后，我们就可以通过<用户名>.github.io来访问我们的站点了。

Gitee 仓库

  • 创建一个仓库，设置为公开或私有都可以。
  • 仓库名如果设置成和个人空间地址一样，那么访问地址就是： <个人空间地址名>.gitee.io
  • 个人空间地址可在个人资料中查看。我的个人空间地址是tangzhixiong，那么我的博客地址就是：tangzhixiong.gitee.io
  • 仓库名如果设置成别的，那么访问地址则是：<个人空间地址名>.gitee.io/仓库名
  • 例如我把仓库名设置为myblog，那么我的博客访问地址就是：tangzhixiong.gitee.io/myblog
  • 创建完成后，我们需要在该仓库页面中 点击“服务” > Gitee Pages > 勾选“强制使用HTTPS” > 点击“启动”即可。

使用本地代理加速 git 和 npm

由于某些东西经常抽风，在国内有时候 Git 连接 Github 和 Node.js 使用 NPM 包管理器变得不那么容易，但是如果我们自备小飞机的话就可以让 Git 和 NPM 走本地代理。这里以 SSR 为例。

  • http || https协议
  • SSH协议
  • （可选） 使用 cnpm 替代 npm 进行加速

搭建本地博客环境

GitHub Page 可以免费托管网页等文件，而且不限容量，是一个不错的选择。有了托管平台，只需一个博客生成器即可完成博客的搭建。选择了 hexo，主题采用 NexT。
